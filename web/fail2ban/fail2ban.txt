✅ ✅ Structure des fichiers Fail2Ban
Tout est dans :
/etc/fail2ban/

👉 voici les dossiers importants :
/etc/fail2ban/
├── jail.conf        ❌ NE PAS TOUCHER
├── jail.local       ✅ ton fichier principal
├── jail.d/          ✅ configs séparées (optionnel)
├── filter.d/        ✅ regex (filtres)
├── action.d/        ✅ actions (ban/unban)
├── fail2ban.conf    ❌ global (rarement touché)

👉 c’est la base [yorkhost.fr]

✅ ✅ Où TU dois mettre tes configs
🔹 1. Configuration principale
👉 fichier :

tes jails (nginx, ssh…)

✅ toujours utiliser .local

✅ ✅ Ordre de lecture (important 🔥)
Fail2Ban lit :
jail.conf       (base)
jail.d/*.conf
jail.local      (override)
jail.d/*.local

👉 donc :
✅ .local écrase .conf [deepwiki.com]

✅ ✅ 2. Filtres (le cœur du système)

🔹 404 scan (bots / scanners)
👉 fichier :
Shellsudo nano /etc/fail2ban/filter.d/nginx-404.confAfficher plus de lignes
INI[Definition]failregex = ^<HOST> - .* "(GET|POST|HEAD).*" 404ignoreregex =Afficher plus de lignes
👉 détecte :

scan de vulnérabilités
crawling agressif


🔹 abus API (très important pour FastAPI 🔥)
Shellsudo nano /etc/fail2ban/filter.d/nginx-api-abuse.confAfficher plus de lignes
INI[Definition]failregex = ^<HOST> - .* "(GET|POST).* /api/.*" (429|403)ignoreregex =Afficher plus de lignes
👉 détecte :

dépassement rate limit nginx
accès interdit API


🔹 bots / user-agents suspects
Shellsudo nano /etc/fail2ban/filter.d/nginx-badbot.confAfficher plus de lignes
INI[Definition]failregex = ^<HOST> - .* "(GET|POST).*" .* "(curl|wget|python-requests|bot|scanner)"ignoreregex =Afficher plus de lignes
👉 bloque :

scraping simple
bots malveillants
